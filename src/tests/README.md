# 测试入口说明

本目录包含三个独立的测试入口点：

## 1. test_game.py - 完整游戏测试 🎮
测试完整的游戏功能，包括地图、玩家移动、怪物系统等。

**运行：**
```bash
python test_game.py
```

**控制：**
- WASD / 方向键：移动
- 鼠标：瞄准
- 左键：射击
- ESC：退出

**测试内容：**
- 地图系统和渲染
- 玩家移动和碰撞
- 摄像机跟随
- 怪物生成和AI
- 子弹系统
- UI显示

---

## 2. test_inventory.py - 背包系统测试 🎒
独立测试背包和装备系统。

**运行：**
```bash
python test_inventory.py
```

**功能：**
- 生成不同品质的装备（普通/精良/史诗/传奇）
- 拖拽装备
- 空格键旋转装备
- 右键菜单（摧毁/精炼）
- 实时属性计算

**测试内容：**
- 装备生成逻辑
- 网格放置系统
- 属性计算
- UI交互

---

## 3. test_monsters.py - 怪物系统测试 👾
命令行工具，用于测试怪物生成逻辑。

**运行：**
```bash
python test_monsters.py
```

**使用：**
输入游戏天数，系统会生成相应数量和强度的怪物，并显示详细信息。

**测试内容：**
- 怪物生成算法
- 怪物属性计算
- 精英怪物系统
- 怪物技能系统

---

## 注意事项

1. 所有测试都需要在 `src/tests/` 目录下运行
2. 确保已安装pygame和numpy：`pip install pygame numpy`
3. 字体文件缺失会使用系统默认字体
4. 测试程序之间相互独立，可以单独运行

## 开发建议

- 修改游戏核心逻辑：使用 `test_game.py` 进行测试
- 调整装备系统：使用 `test_inventory.py` 验证
- 平衡怪物强度：使用 `test_monsters.py` 检查数值
